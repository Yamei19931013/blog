浏览器兼容问题与解决方案

css
1、不同浏览器默认的margin、padding不同：
清除默认样式 *{margin:0;padding:0}
2、@media screen 、hack
3、内联元素相连之间存在间隔：
1）多个标签写在一行
2）将要闭合的标签和开始标签地方重合
3）在父级上写：font-size：0
js
日期格式的转换

性能优化
1、减少http请求次数
2、当要设置的样式很多试，设置className,而不是直接操作style
3、少用全局变量
4、避免使用css表达式 css Expression
5、图片预加载，将样式表放在底部
6、避免在页面主体中使用table,table要等其中的内容完全下载后才会显示出来

页面从输入url到页面加载显示全过程
1、浏览器根据请求的url交给DNS进行域名解析，找到真实ip,然后像服务器发起请求
2、服务器交给后台处理完成后返回数据，浏览器接受文件（html、js、css、图像等）
3、浏览器对加载到的资源进行语法解析，建立内部数据结构
4、载入解析到资源文件，渲染页面，完成。

二、2.0
关联文档 https://confluence.yunxuetang.com.cn/pages/viewpage.action?pageId=30841565
1.代码接入
NPM安装神策sdk： npm install sa-sdk-javascript
初始化SDK
import sensors from ‘sa-sdk-javascript’
import { domainBaseUrl } from ‘@/configs/domain’
// 神策初始化
sensors.init({
server_url: domainBaseUrl(‘sensors’), // 神策api地址
heatmap: {
// 是否开启点击图，默认为 default 表示开启，自动采集 $WebClick 事件，可以设置 ‘not_collect’ 表示关闭
// 需要 JSSDK 版本号大于 1.7
clickmap: ‘default’,
// 是否开启触达注意力图，默认为 default 表示开启，自动采集 $WebStay 事件，可以设置 ‘not_collect’ 表示关闭
// 需要 JSSDK 版本号大于 1.9.1
scroll_notice_map: ‘not_collect’,
// 不走冒泡的模式，防止click.stop时记录不到点击事件
useCapture: true
},
show_log: process.env.NODE_ENV !== ‘production’ // 产线环境关闭log日志
})

登录，将神策埋点与我们平台的用户关联起来
const userId = window.localStorage.userId
if (userId) {
// 标识真实用户
sensors.login(userId, function () {
// 更新用户平台环境
sensors.setProfile({
platform: ‘xuanxing’,
terminal: ‘PC’, // PC、’Android’ 、 ‘IOS’
})

})
}
神策api配置
sensors: {
dev: ‘https://sa-api.yxt.com/sa?project=default', // 神策测试接收数据服务器
stg: ‘https://sa-api.yxt.com/sa?project=default', // 神策测试接收数据服务器
prod: ‘https://sa-api.yxt.com/sa?project=production' // 神策产线接收数据服务器
}

2.埋点
页面的点击事件神策会自动收集
路由跳转的时候需要增加埋点
router.afterEach((route, from) => {

// 神策记录访问日志
Vue.nextTick(() => {
sensors.quick(‘autoTrack’, {
platform_type: ‘xuanxing’,
terminal_type: ‘PC’, // PC、’Android’ 、 ‘IOS’
page_id: route.meta.page_id,
user_position: window.getLocalStorage(‘positionList’),
user_detp: window.getLocalStorage(‘parttimeDeptList’),
user_role: ‘’
})
})
})
其他根据具体业务需求来埋点

3.验证
页面路由跳转或者点击按钮都会发送记录请求
Console内也会打印日志的具体参数

80端口被system占用解决过程

1、打开RegEdit:开始-运行-输入regedit-调出注册表；　　
2、找到HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\HTTP；　　
3、找到一个DWORD值Start，将其改为0；　　
4、重启电脑，System进程将不会占用80端口。
